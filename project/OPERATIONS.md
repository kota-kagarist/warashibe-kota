# 運営マニュアル

Version 1.0 — 2026-07-14

この文書は運営者向けの標準手順です。公開判断は `RULES.md`、個人情報の境界は `PRIVATE_OPS.md` を優先します。

## 1. ステータス

| Status | 意味 | 公開上の動き |
|---|---|---|
| Preparing | 次の交換品を準備中 | 写真・状態・募集条件を整える |
| Open | 提案受付中 | Issue Formを有効化し締切を表示 |
| Review | 提案確認中 | 新規提案を止め、質問と比較を行う |
| Provisional | 仮合意 | 採用候補と理由を公開し非公開確認へ移る |
| Shipping | 発送・受取中 | 個人情報を公開せず進捗だけ更新 |
| Verifying | 到着確認中 | 状態確認と問題の有無を確認 |
| Completed | 交換完了 | 記録・JSON・サイトを更新 |
| Paused | 一時停止 | 停止理由と再開条件を公開 |
| Declined | 見送り | 見送り理由を公開しIssueを閉じる |
| Cancelled | 中止 | 中止時点と公開可能な理由を記録 |

同時に `Open` 以上へ進める交換は1件だけとします。

## 2. 標準スケジュール

- 募集開始時: サイト、`project.json`、Announcementを更新
- 受付確認: 提案から原則3日以内
- 初期審査: 提案から原則7日以内
- 募集期間: 原則7日、延長は1回
- 追加質問への回答期限: 原則7日
- 仮合意後の発送: 原則3営業日以内
- 到着確認: 到着から48時間以内
- 非公開情報の削除: 完了・中止から30日以内。争いがある場合は解決後30日以内、最長90日を目安

期限は義務ではなく運営目安です。遅れる場合はIssueへ短い進捗を残します。

## 3. 募集開始チェックリスト

- [ ] 所有権に問題がない
- [ ] 名称、型番、付属品を確認した
- [ ] 傷、不具合、使用歴を記載した
- [ ] 動作確認を実施した、または未確認と明記した
- [ ] 写真から顔、住所、ラベル、位置情報を除いた
- [ ] 画像のEXIFを削除した
- [ ] 梱包サイズと配送可否を確認した
- [ ] 交換不可品に該当しない
- [ ] 募集開始日と締切を決めた
- [ ] `docs/data/project.json` を更新した
- [ ] 交換記録ファイルを作成した
- [ ] 公開データ検査を通した

## 4. 提案のトリアージ

提案Issueを受けたら、次の順に確認します。

### Gate A — 公開安全性

- 個人情報がない
- 写真に配送ラベル、顔、住所、生活圏が写っていない
- 不審な外部リンクや追跡URLがない

問題があれば、内容を引用せず修正依頼を出します。重大な個人情報は公開Issueで指摘せず、削除対応を優先します。

### Gate B — 対象可否

- 18歳以上・国内交換である
- 現物の所有品である
- 交換不可品に該当しない
- 配送または安全な手渡しが可能である

### Gate C — 情報充足

- 名称・型番
- 使用期間・入手時期の目安
- 傷・欠品・不具合
- 動作確認
- 付属品
- おおよその寸法・重量
- 交換理由と次の使い道

不足があれば `status:review` とし、質問します。

## 5. 選定手順

募集締切後、各提案について次を短く公開します。

- 次の使い道: Strong / Acceptable / Weak
- 状態の透明性: Clear / Needs confirmation / Insufficient
- 安全と物流: Feasible / Conditional / Not feasible
- 次の交換可能性: High / Medium / Low
- 物語性: Strong / Moderate / Limited

価格差は補足情報にとどめ、採否の主理由にしません。採用理由と、他提案を見送った主な理由を公開します。提案者同士を順位付けしたり、人格を評価したりしません。

## 6. 仮合意チェックリスト

- [ ] 採用候補をIssueで宣言した
- [ ] モノ、状態、付属品を再確認した
- [ ] 双方が発送前撤回可能であることを確認した
- [ ] 送料は各自発送分を負担すると確認した
- [ ] 発送期限を確認した
- [ ] 追跡可能な配送方法を確認した
- [ ] 非公開連絡手段へ移る合意を得た
- [ ] 公開記録に残す名称を確認した

この時点ではまだ `Completed` にしません。

## 7. 非公開連絡とデータ管理

- 共有するのは発送に必要な最小限の情報だけ
- 個人情報をスクリーンショット、GitHub、Notion、公開クラウドへ転記しない
- 追跡番号を公開Issueへ貼らない
- 非公開情報は一時保存場所を一つに限定する
- 完了・中止後30日以内に削除し、削除日だけを運営記録へ残す
- 争いがある場合のみ解決まで保持し、解決後30日以内に削除する

## 8. 発送前チェックリスト

- [ ] 初期化・ログアウト・アクティベーション解除を完了した
- [ ] SIM、SDカード、記録媒体、私物を取り除いた
- [ ] シリアル番号を控えた。公開時は必要に応じて一部を伏せた
- [ ] 最終動作確認の写真・動画を保存した
- [ ] 付属品を並べた写真を保存した
- [ ] 梱包前・梱包後の写真を保存した
- [ ] 箱内で動かないことを確認した
- [ ] 配送禁止物に該当しないことを確認した
- [ ] 追跡可能な方法で発送した

## 9. 到着・完了

双方が次を確認します。

- 外装の大きな破損
- 届いた物と合意内容の一致
- 付属品
- 申告された状態との一致
- 基本動作

問題がなければIssueへ「双方確認済み」とだけ記録し、次を更新します。

1. `project/exchange-records/NNN-*.md`
2. `docs/data/exchanges.json`
3. `docs/data/project.json`
4. サイト写真・説明
5. 決定記録が必要な場合は `project/decisions/`

Pull RequestでPrivacy Checkを行い、マージ後にIssueを閉じます。

## 10. 問題発生時

### 個人情報が公開された

1. まず公開範囲を止める・削除する
2. 内容を別の公開コメントへ転載しない
3. 必要ならGitHub Supportへの削除依頼を検討する
4. 原因と再発防止だけを匿名で記録する

### 説明と状態が大きく違う

1. 到着後48時間以内に非公開で写真を共有
2. 何が合意と違うかを事実で整理
3. 継続、返送、代替対応を協議
4. 公開Issueには個人情報を除いた結論だけ記録

### 配送事故

1. 発送者が配送会社へ連絡
2. 梱包写真、追跡情報、到着時写真を保全
3. 調査中は `Paused` とする
4. 解決後、公開可能な範囲で結果を記録

### 連絡途絶・不審行為

- 7日連絡がない場合は一度リマインド
- さらに7日連絡がなければ中止可能
- 脅迫、個人特定、詐欺の疑い、安全上の不安がある場合は期限を待たず停止

## 11. ルール変更

- 軽微な文言修正: Pull Requestで変更
- 交換条件、対象範囲、費用負担、個人情報保持期間の変更: Decision Recordを追加
- 進行中の交換へ不利益な変更を遡及適用しない
- ルールのVersionと更新日を更新する
